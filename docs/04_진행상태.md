## 한 줄 요약
엑셀 `업로드.xlsm`(귀금속 제조공정 중량/로스/입출고 시스템)을 Next.js+Supabase+Vercel(무료)로 이전 중.
골격·DB·작성 입력·탭 네비·다크모드까지는 동작. **공정 화면의 핵심 액션 로직(완료/집계/투입/이관/분할)은 아직 원본과 불일치 — 다음에 하나씩 재검증 필요.**

## 실행/환경
- 앱: `web/` (Next.js 16, App Router). `cd web && npm run dev` → http://localhost:3000
- `web/.env.local` 에 Supabase URL + publishable(anon) key 설정됨(사용자 로컬).
- Supabase 클라우드 무료 프로젝트. SQL 적용 순서:
  1. `supabase/migrations/0001_init.sql`
  2. `supabase/seed.sql`
  3. `supabase/migrations/0002_locked.sql`  ← lots.locked 컬럼 (필수)
  4. 임시 RLS 허용(로그인 전 확인용): `*_anon_all` 정책 (이미 적용됨, 로그인 슬라이스에서 잠글 것)

## ★ 미해결: 공정 액션 로직 (다음 세션 최우선)
사용자 피드백 — "로직이 다 잘못되어 있다, 하나씩 잡아야 함".
원본 VBA와 1:1 재대조하며 검증할 대상:
- **완료 처리/집계** (Module7 集計): 작업중→완료 전이. 일련번호 유지/그룹화, 작업전=입중량, 작업후/실중량 수동.
  → 현재 구현(`completeLots`)이 실제 워크플로우(어느 버튼/순서/대상열)와 맞는지 재확인.
- **투입**(Module4, 작업중 A:K→대상) vs **이관**(Module9, 완료 M:Z→대상 B:K) 컬럼 매핑 정확성.
- **출고**(Module10 출고_선택행_복사_공용), **타부서투입**(Module16) 아직 미구현.
- 분할(Module1/11 SplitNumberByCount), 삭제/위로올림(Module6/8/23/24/32) 재확인.
- io 계열(기계/검수)의 입고→출고 전이가 work와 어떻게 다른지(실중량/Tag수정/Tag중량/Tag로스/출고중량) 원본 확인.

### 재검증 방법(권장)
- `_vba/`에 디컴파일된 86개 모듈 원본 있음. 기능별로 한 개씩:
  1. 해당 VBA Sub의 입력(선택 범위)·출력(대상 열)·부수효과(색칠/잠금/로그) 정독

  3. 그 다음에 web 액션(`web/src/app/process/[id]/actions.ts`) 수정
- 즉, 코드 먼저 짜지 말고 **VBA 정독 → 사용자 확인 → 구현** 순서.
