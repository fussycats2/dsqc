# 업로드.xlsm 완전 분석서

> 작성일: 2026-05-29
> 대상 파일: `업로드.xlsm` (5.4MB, .xlsm 매크로 통합문서)

## 1. 한 줄 요약

**귀금속(금/주얼리) 제조 공정의 "중량 추적 + 로스(loss) 관리 + 입출고 장부" 시스템**이다.
금은 귀금속이라 공정마다 작업 전/후 중량을 기록해 손실(로스)을 정산하며, 각 작업 단위에
`부서약자_날짜_순번` 형식의 일련번호를 부여해 공정 간 흐름을 추적한다.

**소량 운영 데이터**. (DB 용량 부담 거의 없음)

## 3. 시트(공정) 구성 — 44개

- **메뉴**: 동적 버튼 네비게이션 (카테고리 필터 + 시트 점프)
- **작성**: 데이터 입력 허브 (주문/내역 작성)
- **일반 공정(18K)**: 기계 · 양장 · 캐스팅 · 개발 · 컷팅
- **14K 공정**: 조립14K · 캐스팅14K · 컷팅14K
- **연마**: 연마(조립) · 연마(캐스팅) + 각 14K
- **뻥**: 뻥(기계/양장/캐스팅/개발) + 14K
- **빠우(버프/광)**: 양장볼 · 기계 · 할로우 · 패션반지 · 캐스팅양장 · 캐스팅체인 ·
  초광(조립/캐스팅) · 개발 + 각 14K (가장 세분화된 공정군)
- **검수**: 검수(기계/볼/양장/캐스팅) + 14K
- **raw**: 전 공정 집계 (INDIRECT+SUM으로 각 시트 합산, 중량오류 검출)
- **sum**: 대시보드 (파트별/검수 입출고 현황, K18/K14 이월)
- **입출로그**: 감사 로그 (모든 입·출고 이력)

### 공정 분류(메뉴 카테고리)

14K 계열은 파란색으로 구분 표기. 검수 공정은 일련번호 접두어가 `Q*`.

## 4. 데이터 스키마 (공정 시트 공통)

각 공정 시트는 **12행=헤더, 13행부터 데이터**. 좌측=입고 블록, 우측=출고 블록.

### 일반 공정 시트 (예: 기계, 검수)

**입고(좌)**
| 열 | A | B | C | D | E | F | G | H | I | J | K |
|----|---|---|---|---|---|---|---|---|---|---|---|
| 의미 | 일련번호 | 내역 | 수량 | 중량 | Tag | Q | 납기 | 원중량 | 비고 | 투입시간 | 투입부서 |

**출고(우)**
| 열 | L | M | N | O | P | Q | R | S | T | U | V | W | X | Y |
|----|---|---|---|---|---|---|---|---|---|---|---|---|---|---|
| 의미 | 일련번호 | 내역 | 수량 | 실중량 | Tag | Q | 납기 | 원중량 | 비고 | 이전파트 | Tag수정 | Tag중량 | Tag로스 | 출고중량 |

- `Y(출고중량) = IF(O=0,"",IF(X=0,O,O+P-W))` — 실중량 + Tag - Tag중량
- 11행 = 블록 합계 `SUM(…13:…1000)`

### 공정 스키마는 두 계열로 나뉜다 (★중요)

**계열 A — 입고/출고형 (`io`)**: 일반 공정(기계·양장·캐스팅·개발·컷팅·조립14K·캐스팅14K·컷팅14K)
**+ 검수 전부**(검수 기계/볼/양장/캐스팅/14K) — 검수 헤더는 일반 공정과 **완전 동일**(제목·SUM범위만 다름)
- 입고(A~K): 일련번호·내역·수량·중량·Tag·Q·납기·원중량·비고·투입시간·투입부서
- 출고(L~Y): 일련번호·내역·수량·**실중량(O)**·Tag·Q·납기·원중량·비고·이전파트·**Tag수정(V)·Tag중량(W)·Tag로스(X)·출고중량(Y)**
- 로스 개념 = **Tag 기반 보정**: `출고중량 Y = 실중량 O + Tag P − Tag중량 W`, 로스는 Tag로스(X)

**계열 B — 작업중/완료형 (`work`)**: **연마 · 뻥 · 빠우** (전 변형)
- 작업중(B~L): 일련번호·내역·수량·**입중량(E)**·Tag·Q·납기·원중량·비고·중량(K)·이전파트(L) (A=완료마킹/현황)
- 완료(M~Y): 일련번호·내역·수량·**작업전(P)·작업후(Q)·로스(R)·로스율(S)**·Tag·Q·납기·원중량·비고·이관시간(Y)
- 로스 개념 = **가공 중량 손실**: `로스 R = 작업전 P − 작업후 Q`, `로스율 S = 1 − 작업후/작업전`. Tag보정 칸 없음.

> 정정 메모: 사용자 확인 결과 **연마/뻥/빠우만 동일(계열 B)**, **검수는 일반 공정과 같은 계열 A**.
> → 웹앱 `processes.schema_type ∈ {io, work, entry}` 로 구분. 저장 테이블은 공통(`lots`)이되 사용 컬럼·UI·로스식이 계열별로 다름.

### 작성 시트 (입력)

`내역·수량·중량·Tag·Q·납기·원중량(수리)·비고` + **Tag 검증**:
`K = IF(ROUNDDOWN(수량*0.035,2)=Tag, "OK","NG")`
→ **핵심 규칙: 1 tag = 0.035g**

### 입출로그 시트

`날짜시간 · 작업유형(입고/출고/투입/이관…) · 원본시트 · 대상시트 · 시작열 · 붙여넣기행 · 내역 · 수량 · 중량 · Tag · Q · 납기 · 원중량 · 비고`

## 5. 일련번호 체계 (핵심 식별자)

형식: **`약어_YYMMDD_순번3자리`** (예: `M_260521_001`)

| 공정 | 약어 | | 공정 | 약어 |
|------|------|---|------|------|
| 기계 | M | | 검수(기계) | QM |
| 양장 | Y | | 검수(볼) | QB |
| 캐스팅 | C | | 검수(양장) | QY |
| 개발 | G | | 검수(캐스팅) | QC |
| 컷팅 | T | | 검수(조립)14K | QA(파랑) |
| 조립14K | A(파랑) | | 검수(캐스팅)14K | QC(파랑) |
| 캐스팅14K | C(파랑) | | | |
| 컷팅14K | T(파랑) | | | |

- 순번: **같은 약어+날짜 prefix 내 최대 순번 +1** (일별 리셋)
- 접미 분할 표기 가능: `M_260520_017-2-1` (수량 분할 시 `-n` 부여)
- 집계 시 묶음 표기: `T_260520_(003,004)`

## 7. 핵심 워크플로우 (데이터 흐름)

```
[작성] 주문/내역 입력 (Tag검증)
   │  보내기(입고)
   ▼
[공정 시트 입고블록]  ──투입/이관──▶ [다음 공정]  ──…──▶ [빠우/연마]
   │ (일련번호 생성)                      (로스 기록)
   │  출고블록 기록 (Tag로스/출고중량)
   ▼
[검수] 최종 검수 → 출고
   │
   ▼
[raw] 전 공정 SUM 집계 → [sum] 대시보드 / 중량오류 경고
   │
   ▼
[입출로그] 모든 입·출고 감사기록
   │  월마감
   ▼
[백업본 저장 + 초기화] / [품질결산서 전송]
```

## 8. 핵심 비즈니스 규칙 (이식 필수)

1. **1 Tag = 0.035g** — Tag 검증/보정의 기준 상수
2. **출고중량 = 실중량 + Tag − Tag중량** (`Y=O+P-W`)
3. **로스 = 작업전 − 작업후**, 로스율 = `1 − 작업후/작업전`
4. **일련번호 = 약어_YYMMDD_일별순번**, 14K는 색 구분
5. **수량 분할** 시 `-n` 접미, **집계** 시 prefix 그룹 + 괄호 묶음
6. **중량오류 검출**: raw 정산 영역 합계 ≠ 0 이면 경고 (입고중량 vs 출고+로스 불일치)
7. **공정 흐름**: 이전파트(U열) 기록으로 추적성 유지
8. **상태**: A열="완료" 마킹 + 배경색으로 처리완료 표시

## 9. 기술 부채 / 리팩토링 동인

- 시트가 곧 테이블 = 동일 스키마 44벌 복제. 신규 공정 추가 = 시트+버튼+VBA 수동 복제
- `보내기__기계`처럼 **공정마다 래퍼 Sub** 수백 개 (하드코딩)
- `INDIRECT` 집계 = 느리고 깨지기 쉬움, 동시편집 불가
- 단일 .xlsm 파일 = 동시 사용 불가, 버전관리·백업 수동
- 매크로 보안 경고, 엑셀 의존, 모바일/원격 불가
- 인쇄/페이지셋업 로직이 비즈니스 로직과 뒤섞임
