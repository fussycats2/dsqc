# 셋업 가이드 — 첫 수직 슬라이스 실행

현재 구현된 것(슬라이스 1):
- Next.js 16(App Router, TS, Tailwind) 앱 = `web/`
- DB 스키마 마이그레이션 = `supabase/migrations/0001_init.sql`
- 공정 마스터 40개 시드 = `supabase/seed.sql` (`tools/gen_seed.py`로 .xlsm에서 생성)
- 화면: `/` 공정 목록(카테고리별) · `/process/[id]` 입고/출고 그리드 + **입고 1건 추가**(일련번호 자동 생성)

## 1. Supabase 무료 프로젝트 만들기 (사용자)

1. https://supabase.com → 로그인 → **New project** (Region: Northeast Asia(Seoul) 권장, 무료 플랜)
2. 프로젝트 생성 후 **SQL Editor**에서 순서대로 실행:
   - `supabase/migrations/0001_init.sql` 전체 붙여넣기 → Run
   - `supabase/seed.sql` 전체 붙여넣기 → Run
3. **Project Settings → API**에서 두 값 복사:
   - `Project URL`
   - `anon public` key

> 단일 공용 계정: **Authentication → Users → Add user**로 직원 공용 계정 1개 생성
> (이메일/비밀번호). RLS가 `authenticated`만 허용하므로 로그인 화면은 슬라이스 2에서 추가.
> 지금 데이터를 바로 보려면 임시로 SQL Editor에서 테스트하거나 슬라이스 2의 로그인까지 기다리면 됩니다.

## 2. 로컬 실행

```bash
cd web
cp .env.local.example .env.local
#  .env.local 에 위에서 복사한 URL/anon key 입력
npm run dev
#  http://localhost:3000
```

`.env.local` 미설정 시 홈 화면이 안내 배너를 표시합니다(앱은 정상 구동).

## 3. Vercel 배포 (무료 Hobby)

1. 이 저장소를 GitHub에 push (루트가 아니라 `web/`가 앱 루트)
2. Vercel → New Project → 저장소 선택 → **Root Directory = `web`**
3. Environment Variables에 `NEXT_PUBLIC_SUPABASE_URL`, `NEXT_PUBLIC_SUPABASE_ANON_KEY` 추가 → Deploy
